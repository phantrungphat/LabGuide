Câu 1: Những vị trí nào có thể đặt Javascript trong file HTML và dặt theo cú pháp nào? Nên đặt Javascript tại vị trí nào để có hiệu suất tốt nhất? Cho ví dụ minh họa.

Đầu tiên, bạn có thể đặt đoạn mã Javascript bất cứ đâu trong tập tin HTML và bất cứ khi nào bạn muốn!. Bạn có thể:

  • Tại thẻ <head>...</head>
  • Tại thẻ <body>...</body>
  • Cả thẻ <head>...</head> và <body>...</body>

Cũng như các ngôn ngữ lập trình khác, Javascript sẽ chạy biên dịch từ trên xuống dưới và từ trái qua phải. Chính vì vậy, nếu bạn sử dụng một hàm mà phía trên nó không tồn tại thì hàm đó sẽ bị báo lỗi undefined. Vì vậy, chúng ta cần cân nhắc cho việc đặt đoạn mã Javascript của chúng ta ở đâu thì sẽ hợp lý. Ngoài ra, để giải quyết trường hợp này chúng ta còn có thể sử dụng sự kiện onload trong Javascript.

Về phần cú pháp thì đã được nhắc đến tại Task3.1. Có 3 cách: Internal, External, và Inline. Mặc dù cho chúng ta đặt như thế nào thì đều phải đảm bảo rằng đoạn mã Javascript của chúng ta phải nằm trong cặp thẻ <script>...</script>.

Ở thời kỳ HTML5 chưa ra đời, có thể nói vị trí đặt Javascript tốt nhất đó là cuối trang và trước thẻ body. Bởi vì các bạn cứ tưởng tượng, nếu chúng ta đặt ở đầu trang, mã HTML đang tải và phải dừng lại để chờ tải về các tập tin Javascript được tham chiếu vào trang, quá trình này làm chậm hoặc thậm chí làm block trang web nếu nó nghiêm trọng. Chính vì thế, đặt đoạn mã Javascript ở cuối là đúng đắn.

Tuy nhiên, khi HTML5 ra đời đã giải quyết vấn đề này, hai thuộc tính async và defer được giới thiệu để sử dụng cho thẻ <script>. Sử dụng hai thuộc tính này sẽ giúp mã lệnh JavaScript có thể được tải bất đồng bộ cùng với mã HTML. Điều này giúp cho trình duyệt có thể đồng thời tải cả mã HTML và JavaScript cùng một lúc.

Khi đó bạn nếu bạn sử dụng thẻ <script> ở bên trong thẻ <head> như sau:

				</html>
				    <head>
				        <script src="/js/main.js" type="text/javascript" async></script>
				    </head>
				<body>
				...
				</body>
				</html>

Tốc độ tải trang sẽ nhanh hơn khi để thẻ <script> ở cuối trang (trước thẻ <body>) vì lúc này chúng ta tận dụng được lợi thế trình duyệt tải về đồng thời HTML và JavaScript cùng một lúc.

Câu 2: Trong Javascript có những kiểu dữ liệu nào? Làm thế nào để tạo một biến để lưu trữ giá trị tương ứng với những kiểu dữ liệu đó?

Javascript có các kiểu dữ liệu Dynamic, nghĩa là việc kiểm tra kiểu được thực hiện trong lúc runtime thay vì compile time. Với ngôn ngữ được nhập động, một biến có cùng tên có thể được sử dụng để giữ các loại dữ liệu khác nhau.

Ví dụ ta có thể định nghĩa một biến a được gán để giữ các kiểu dữ liệu khác nhau hoặc khởi tạo nhưng không xác định.

								
							var a = 1998; 	//a là kiểu number
							var a = "Ahihi"; 	//a là kiểu string
							var a = true; 	//a là kiểu boolean
							var a; 	//a là kiểu undefined

Chúng không cần phải khai báo 1 cách rõ ràng với 1 kiểu dữ liệu trước khi chúng sử dụng.

Kiểu dữ liệu Number

Kiểu dữ liệu này dành cho cả số nguyên và số có dấu chấm động (integer và float). Nếu một số quá lớn chúng ta có thể sử dụng dạng mũ để biểu diễn, vd như: var x = 1998e-5; ... Các số luôn luôn là 64 bit và kiểu float.

Có thể biểu diễn giá trí của Number dạng nhị phân, thập phân, thập lục phân,...

							var x = 0xFF;             // x có giá trị bằng 255

Kiểu dữ liệu String

String là một tập hợp gồm các ký tự, chúng được viết bên trong cặp dấu nháy kép hoặc cặp dấu nháy đơn.

Trường hợp trong chuỗi cũng có xuất hiện dấu nháy đơn hoặc nháy đôi thì bắt buộc bạn phải thêm ký tự \ đằng trước dấu nháy đó nếu không sẽ bị lỗi cũ pháp.

							var x = "Ahihi con \"khỉ\" già";

Nếu chúng ta muốn nối chuỗi sử dụng dấu +.

							var x = "Ahihi" + "Đồ ngốc";		// Kết quả là chuỗi "Ahihi Đồ ngốc"

Kiểu dữ liệu Booleans

Kiểu dữ liệu Boolean có thể là một trong hai giá trị, true hoặc false Booleans được sử dụng để biểu diễn các giá trị luận lý được liên kết với nhánh logic của toán học, thông báo các thuật toán trong khoa học máy tính.

Kiểu dữ liệu Arrays

Mảng là một tập hợp các phần tử lại và mỗi phần tử sẽ được đánh dấu một vị trí trong tập hợp đó. Trong javascript nếu mảng có 10 phần tử thì các phần tử sẽ được đánh dấu từ 0 -9.

Để khai báo array, chúng ta có thể sử dụng từ khóa new Array() hoặc sử dụng cặp dấu ngoặc vuông [].

							var name_array = new Array(1,2,3);
							// hoặc
							var name_array = [1,2,3];

Kiểu dữ liệu Objects

Như chúng ta đã biết, Javascript không phải là một ngôn ngữ hướng đối tượng, nhưng nhờ vào kiểu dữ liệu Object thì chúng ta hoàn toàn có thể lập trình hướng đối tượng được.

Mỗi Object sẽ có các thuộc tính và phương thức.

Thuộc tính là những đặc điểm (có thể hiểu là biến) cần lưu trữ trong đối tượng. Ví dụ ta tạo đối tượng Sinh viên thì có thể có thuộc tính là Mã số sinh viên, họ tên, năm ra trường,...

Cách khai báo:
Cách 1: Sử dụng từ khóa new Object()

							// Khởi tạo
							var Sinhvien = new Object();
							 
							// Thêm thuộc tính
							Sinhvien.tuoi = '';
							Sinhvien.namratruong = '';
							Sinhvien.hoten = '';
Cách 2: Sử dụng từ khóa {} và thêm thuộc tính ngay lúc khai báo 
							// Khởi tạo
							var Sinhvien = {
							Sinhvien.tuoi = '';
							Sinhvien.namratruong = '';
							Sinhvien.hoten = '';
							}
Cách 3: Sử dụng từ khóa {} và thêm thuộc tính sau đó 
							// Khởi tạo
							var Sinhvien = {};
							Sinhvien.tuoi = '';
							Sinhvien.namratruong = '';
							Sinhvien.hoten = '';

Phương thức là những hành động (có thể hiểu là hàm) của đối tượng. Ví dụ đối tượng Sinh viên có thể có phương thức là Lưu bài tập, học bài,...

Cách khai báo:

Cách 1: Sử dụng từ khóa new Object() 
							// Khởi tạo
							var Sinhvien = new Object();
							Sinhvien.Hoc = function(){
						    	// do some thing
							};
						 
							Sinhvien.Luu = function(){
						    	// do some thing
							};
Cách 2: Sử dụng từ khóa {} và thêm phương thức ngay lúc khai báo 
							// Khởi tạo
							var Sinhvien = {
								Hoc : function(){
							    	// do some thing
								},			 
								Luu : function(){
							    	// do some thing
								}
							}
Cách 3: Sử dụng từ khóa {} và thêm phương thức sau đó 
							// Khởi tạo
							var Sinhvien = {};
							Sinhvien.Hoc = function(){
							    // do some thing
							};		 
							Sinhvien.Luu = function(){
							    // do some thing
							};

Tất cả đều được khuyến khích nên sử dụng cách 2.

Nếu bạn muốn biết một biến thuộc kiểu dữ liệu gì thì sử dụng hàm typeof.

Câu 3: Làm thế nào để truy xuất (lấy giá trị) trong 1 thẻ HTML bất kỳ và gán vào 1 biến để trong Javascript theo ID, theo Class, theo Name, theo tên thẻ HTML và theo CSS Selector (ví dụ .class, #id)? Cho ví dụ minh họa.

Như đã đề cập khá chi tiết ở Task3.1 - câu 3 chúng ta có thể sử dụng:

  • document.getElementById()
  • document.getElementsByClassName()
  • document.getElementsByName()
  • document.getElementsByTagName()

Sau đó sử dụng innerHTML để lấy nội dung hoặc document.CacPhuongThucOTren().attributeName để lấy giá trị.

Tìm thẻ HTML theo ID

							// Lấy thẻ input
							var element = document.getElementById('idAhihi');
							 
							// Lấy giá trị của thẻ input xuất ra
							document.write(element.value);

Tìm thẻ HTML theo tên của thẻ HTML

							// Lấy thẻ input
							var element = document.getElementsByTagName('idAhihi');
							 
							// Lấy giá trị của thẻ input xuất ra
							document.write(element[0].value);

Tìm thẻ HTML theo tên class

							// Lấy thẻ input
							var element = document.getElementsByClassName('idAhihi');
							 
							// Lấy giá trị của thẻ input xuất ra
							document.write(element[0].value);

Tìm thẻ HTML theo cú pháp của Selector CSS

Thông thường, khi chúng ta chọn các thẻ theo class thì chắc chắn sẽ có một số kết quả chúng ta không mong đợi. Chính vì vậy, Javascript DOM có một phương thức kết hợp với CSS Selector để truy vấn có độ chính xác cao hơn.

Tương tự như kiểu "chỉ ra đường dẫn" cho phương thức thì phương thức sẽ chọn chính xác các đối tượng mà ta nhắm đến. Ví dụ:

							
							<input type="text" value="thẻ không cần lấy" class="website"/>
						    <div>
						        <input type="text" value="thẻ cần lấy" class="website"/>
						        <input type="text" value="thẻ không cần lấy"/>
						    </div>

Câu hỏi đặt ra là làm thế nào có thể chọn đúng một thẻ input nằm trong thẻ div và có class="website"?

Rất đơn giản, chúng ta chỉ ra "đường dẫn" cho phương thức document.querySelectorAll() như sau:

							var element = document.querySelectorAll("div input.website");

Như thế là đã có thể chọn đúng đối tượng mà chúng ta mong muốn.

Câu 4: Xác định các lệnh Javascript tương ứng để thực hiện các yêu cầu:

Viết ghi chú trong Javascript (1 dòng, nhiều dòng)

Nếu nội dung ghi chú nằm gọn trên một dòng thì bạn chỉ cần đặt cặp dấu // phía trước nó.

							
							<script>
								alert("ahihi");
								// Đây là phần ghi chú nè!!!
								alert("ahihi");
							</script>

Nếu phần nội dung chú thích trải dài trên nhiều dòng thì ta dùng cặp dấu /* */ để bao quanh chúng.

							
							<script>
								alert("ahihi");
								/* Đây là phần ghi chú nè !!!
								Thấy ghi chú dài hơm...
								Dài quá trời dài nè... */
								alert("ahihi");
							</script>

Ngoài ra, cặp dấu /* */ cũng có thể dùng để chú thích cho phần nội dung nằm gọn trên một dòng.
